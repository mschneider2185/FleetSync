# FleetSync Brand Identity System

> **Single source of truth for all FleetSync UI development.**
> Every page, component, and screen must reference this document.
> Do not deviate from these specifications without updating this file first.

---

## 1. Brand foundation

### Identity

| Attribute       | Value                                                                 |
|-----------------|-----------------------------------------------------------------------|
| Product name    | FleetSync                                                             |
| Tagline         | Plan. Allocate. Execute.                                              |
| Subtitle        | Operations Platform                                                   |
| Parent brand    | EQT Corporation                                                       |
| Co-brand line   | Powered by EQT                                                        |
| Domain concept  | `app.fleetsync.eqt.com`                                               |

### Voice and tone

- **Authoritative but not stiff.** This is an ops tool used by field teams and leadership alike.
- **Data-first.** Lead with numbers, not adjectives. "193 loads tracked today" beats "Track your loads easily."
- **Concise.** Labels are terse and uppercase. Descriptions are one sentence max.
- **No marketing fluff inside the app.** Save narrative copy for the landing page hero only.

### Audience

| Persona              | Context                                          |
|----------------------|--------------------------------------------------|
| Operations planner   | Daily driver — Gantt, allocation grid, journal    |
| Frac support lead    | Water module, CWF inventory, supply board         |
| Operations leadership| Executive dashboard, hauler scorecard, exports    |
| Viewer (read-only)   | Same screens, no edit capabilities                |

## 2. Color system

### Core palette

All colors defined as CSS custom properties. Use variable names in code — never raw hex values.

```css
:root {
  /* ── Navy foundation ── */
  --fs-navy:          #0a1628;   /* Page background, primary surface          */
  --fs-navy-mid:      #152238;   /* Elevated surfaces, cards, sidebar hover   */
  --fs-slate:         #1e2d45;   /* Tertiary surface, input backgrounds       */

  /* ── EQT Magenta — primary action color ── */
  --fs-magenta:       #E91E78;   /* CTAs, active states, brand accent         */
  --fs-magenta-bright:#FF3D94;   /* Hover states on magenta elements          */
  --fs-magenta-deep:  #C4105F;   /* Pressed states, dark-on-dark contexts     */
  --fs-magenta-glow:  rgba(233, 30, 120, 0.15); /* Tinted backgrounds        */

  /* ── Blue — data and information ── */
  --fs-blue:          #185FA5;   /* Charts (delivered), info badges            */
  --fs-blue-bright:   #378ADD;   /* Hover on blue elements                    */
  --fs-blue-glow:     #85B7EB;   /* Hero accent text, decorative highlights   */

  /* ── Teal — positive and success ── */
  --fs-teal:          #1D9E75;   /* Success states, positive deltas           */
  --fs-teal-light:    #5DCAA5;   /* Positive delta text, "Live" tags          */

  /* ── Amber — warning and in-progress ── */
  --fs-amber:         #EF9F27;   /* Warning states, NPT indicators            */
  --fs-amber-dark:    #BA7517;   /* Warning text on dark backgrounds          */

  /* ── Red — error and conflict ── */
  --fs-red:           #E24B4A;   /* Error states, conflict indicators         */

  /* ── Text ── */
  --fs-text:          #E8ECF2;   /* Primary text on dark backgrounds          */
  --fs-text-muted:    #8A99B4;   /* Secondary text, labels, placeholders      */

  /* ── Borders and surfaces ── */
  --fs-border:        rgba(255, 255, 255, 0.08);  /* Default border           */
  --fs-border-hover:  rgba(255, 255, 255, 0.15);  /* Hover border             */
  --fs-border-active: rgba(255, 255, 255, 0.30);  /* Focus / active border    */
  --fs-surface:       rgba(255, 255, 255, 0.04);  /* Subtle raised surface    */
}
```

### Color usage rules

| Role               | Variable              | Usage example                                |
|---------------------|-----------------------|----------------------------------------------|
| Primary action      | `--fs-magenta`        | Sign-in button, main CTA, active nav item    |
| Action hover        | `--fs-magenta-bright` | Hover state on any magenta element           |
| Tinted background   | `--fs-magenta-glow`   | Active sidebar item bg, "Building" tag bg    |
| Data / info         | `--fs-blue`           | Chart bars (delivered), info indicators      |
| Positive / success  | `--fs-teal-light`     | Positive deltas, "Live" module tags          |
| Warning             | `--fs-amber`          | NPT journal dots, warning badges            |
| Negative / conflict | `--fs-magenta` or `--fs-red` | Conflict count delta, error states    |
| Page background     | `--fs-navy`           | Always. No white backgrounds.                |
| Card / elevated     | `--fs-navy-mid`       | Sidebar, modals, card surfaces, preview bg   |

### EQT magenta integration

The magenta accent (`#E91E78`) is derived from EQT's corporate brand mark — the pink arrow through the Q. In FleetSync it serves as the **primary action color**, replacing the generic blue CTA pattern. Apply it to:

- All primary buttons and CTAs
- Active navigation states (sidebar, tabs, filter pills)
- The "S" in "FleetSync" wordmark
- The "Q" in "EQT" wordmark wherever the co-brand appears
- Section labels (uppercase eyebrow text)
- "Building" status tags
- Hero badge border and dot
- Chart legend for "Committed" data series

**Do not** use magenta for:
- Large background fills (use `--fs-magenta-glow` at 15% opacity instead)
- Body text
- Success states (use teal)
- Warning states (use amber)

## 3. Typography

### Font stack

```css
:root {
  --fs-font-display: 'DM Serif Display', Georgia, serif;
  --fs-font-body:    'Plus Jakarta Sans', -apple-system, BlinkMacSystemFont, sans-serif;
  --fs-font-mono:    'JetBrains Mono', 'Fira Code', monospace;
}
```

**Load from Google Fonts:**
```html
<link href="https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@300;400;500;600;700&family=DM+Serif+Display&display=swap" rel="stylesheet">
```

### Type scale

| Token         | Font               | Size  | Weight | Use case                              |
|---------------|---------------------|-------|--------|---------------------------------------|
| `hero`        | DM Serif Display    | 54px  | 400    | Landing page hero headline only       |
| `display`     | DM Serif Display    | 36px  | 400    | Section titles                        |
| `stat-number` | DM Serif Display    | 32px  | 400    | KPI large numbers, stat bar values    |
| `kpi-number`  | DM Serif Display    | 22px  | 400    | Dashboard KPI card values             |
| `heading`     | Plus Jakarta Sans   | 16px  | 600    | Card titles, sidebar logo text        |
| `body`        | Plus Jakarta Sans   | 14px  | 400    | General body text, descriptions       |
| `body-sm`     | Plus Jakarta Sans   | 13px  | 400    | Module card descriptions, nav links   |
| `label`       | Plus Jakarta Sans   | 12px  | 500    | KPI labels, chart labels              |
| `eyebrow`     | Plus Jakarta Sans   | 11px  | 600    | Section labels — uppercase, 3px tracking |
| `caption`     | Plus Jakarta Sans   | 10px  | 400    | Chart axis labels, legend text        |
| `mono`        | JetBrains Mono      | 12px  | 400    | URL bars, code references, data IDs   |

### Typography rules

- **DM Serif Display** is reserved for numbers and headlines. Never use it for body text or labels.
- **Plus Jakarta Sans** is the workhorse. Use it for everything else.
- **Letter-spacing:** Hero headline = `-1px`. Eyebrow labels = `3px`. Body text = default.
- **Line-height:** Headlines = `1.08–1.2`. Body text = `1.6–1.7`. Labels = `1`.
- **Weight range:** 400 (regular), 500 (medium), 600 (semibold), 700 (bold — logo only).
- **No font-size below 10px.** Ever.

## 4. Logo system

### FleetSync wordmark

The product name is rendered in Plus Jakarta Sans at 20px / weight 600 / letter-spacing -0.5px. The "S" in "Sync" is colored `--fs-magenta`. All other letters are white.

```
Fleet[magenta]S[/magenta]ync
```

- Subtitle "Operations platform" sits below in 10px uppercase, 2.5px letter-spacing, `--fs-text-muted`.

### FleetSync logo mark (icon)

A 40×40px rounded square (`border-radius: 8px`) with `--fs-navy-mid` background and `0.5px solid var(--fs-border)`.

Contents:
- The letters "FS" centered, Plus Jakarta Sans, 16px, weight 700, white.
- A small magenta accent bar: 12×3px, `border-radius: 2px`, positioned at bottom-left (6px from left, 6px from bottom).

**The accent bar must not overlap or cut through the letterforms.** It sits below and to the left as an independent mark.

### EQT co-brand wordmark

When displaying the EQT name:
- Render "EQT" in Plus Jakarta Sans, weight 800.
- The "Q" is colored `--fs-magenta`.
- The "E" and "T" use `--fs-text-muted` (in footer) or white (in co-brand strip).

**No lines, arrows, or slashes through any text.** The magenta Q alone carries the EQT brand accent.

### Co-brand strip

A horizontal bar placed below the hero section:
- `border: 0.5px solid var(--fs-border)`, `border-radius: 10px`, `background: var(--fs-surface)`.
- Left side: EQT wordmark (28px, weight 800, magenta Q).
- Right side (separated by a left border): "Powered by EQT Corporation" in bold, with a one-line descriptor below in `--fs-text-muted`.

## 7. Motion and animation

### Principles

- **Subtle and functional.** No bouncing, no spinning, no gratuitous motion.
- **Fast transitions.** Default duration: `0.2s`. Card hovers: `0.3s`. Chart bars: `0.8s`.
- **Easing:** `ease` for simple transitions. `cubic-bezier(0.23, 1, 0.32, 1)` for chart animations.

### Specific animations

| Element              | Animation                                           |
|----------------------|-----------------------------------------------------|
| Hero badge dot       | `pulse-dot`: opacity 1 → 0.4 → 1 over 2s, infinite |
| Primary button hover | `translateY(-1px)`                                  |
| Module card hover    | Top border width: `0 → 100%` over `0.4s`            |
| Chart bars on load   | Height: `0 → target` over `0.8s`, staggered 100ms per column |

## 9. Chart and data visualization

### Color mapping

| Data series   | Color              | Opacity |
|---------------|--------------------|---------|
| Committed     | `--fs-magenta`     | 0.5     |
| Delivered     | `--fs-blue`        | 1.0     |
| Surplus       | `--fs-teal`        | 0.6     |
| Shortfall     | `--fs-red`         | 0.8     |
| Forecast      | `--fs-blue-glow`   | 0.4     |
| NPT           | `--fs-amber`       | 0.7     |

### Bar charts

- `border-radius: 2px 2px 0 0` (rounded top only).
- Minimum bar height: `4px` (even for zero-ish values, to maintain visual presence).
- Gap between bars in a group: `2px`. Gap between groups: `4px`.

### Axis and labels

- Axis labels: `10px`, `--fs-text-muted`.
- Legend dots: `8×8px`, `border-radius: 2px`.
- Legend text: `10px`, `--fs-text-muted`.

## 10. Iconography

- **Source:** Lucide icons (stroke-based, 24×24 viewBox).
- **Stroke width:** `2` (default).
- **Size in UI:** `16×16px` for nav/sidebar, `20×20px` for module card icons.
- **Color:** Inherits from parent via `currentColor`.

---

## 11. Dark mode note

FleetSync is **dark mode only.** There is no light theme. All specifications in this document assume a dark background. Do not add a light mode toggle or create light-theme variants.

---

## 12. Integration ecosystem footer

When displaying integrated systems, use a horizontal bar with:
- Label: "Integrated with" in `11px`, uppercase, `letter-spacing: 2px`, `--fs-text-muted`.
- System names: `13px`, weight 500, `rgba(138,153,180,0.4)`.
- Current integrations: Databricks, Gemini Tickets, Glancer, Jarvis BI.
