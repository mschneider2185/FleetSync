4) LANE CASCADING: PUSH DOWNSTREAM FRACS WHEN ONE EXTENDS
============================================================

Goal: if a frac extends (end date pushed right), automatically shift subsequent fracs in that same lane so they cannot start until the prior is done.

Important: We only need “push-right cascade” now. Do NOT auto-pull fracs earlier when a job shortens.

Implementation:
- When a schedule is updated (drag in gantt-chart.tsx -> PATCH /api/schedules/:id):
  - Compare oldEnd vs newEnd.
  - If newEnd > oldEnd (deltaDays or deltaMs > 0):
    - Find all other schedules in the same scenario AND same lane that start AFTER the updated schedule start (or are sequenced after it).
    - Shift each downstream schedule by the same delta (preserve duration and existing gaps).
    - Use a DB transaction.
- Make sure conflict detection recomputes based on the new dates.

Edge cases:
- If downstream schedules overlap after shifting (shouldn’t if you shift all by same delta), ensure you don’t create duplicates.
- If a downstream schedule was manually overlapped on purpose, still push it right because the requirement is “if one frac is still going then the next can’t start.”

Acceptance criteria:
- Extend Mingo by +2 days => the next frac in that lane automatically shifts +2 days (and so on downstream).
- Shortening a frac does NOT automatically pull others earlier.
